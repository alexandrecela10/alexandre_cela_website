20% New Users Activation Lift With New Users Onboarding Survey

The Context: We wanted to improve usage retention. So we needed to find the highest leverage point to move the needle.

What were the challenges?

Overall usage retention was quite stable at around 70% month-over-month. But when I dug into specific user segments, I found something interesting.

New users were the problem‚Äîand the opportunity.

Only about 35% of new users came back to use the product in their second month. This was a high-impact area worth investigating.

Finding the Pattern

Through data analysis, I noticed a clear pattern: new users who performed at least one 'advanced feature' in their first month were 2x more likely to return the next month.

This became 2.5x more likely for new users who repeated the usage of such features (‚â•2 uses in the first month).

Important nuance: It could be that highly engaged users naturally find those features, rather than the features causing engagement. But either way, this correlation pointed to a high-leverage intervention worth testing.

The hypothesis: If we can guide new users to discover and use relevant advanced features early, we can improve their retention.

How did I contribute to address this?

My recommendation: Find a way to make it easier for new users to find and use relevant for their use case app advanced features. The PM came up with the new user onboarding survey idea and product. Bonus point, we collect actual users use cases information, which helps for further app and experience personalisation

I collaborated with the product manager on the delivery and experiment design. We structured it as an A/B test with three groups:

What was tested?

  • Control group: Default onboarding (no checklist)
  • Checklist effect: Provide new users with a list of recommended key actions
  • Personalization effect: Provide a personalized list after asking about their use case

I worked closely with the product manager to define the activation metric, design the experiment, and ensure we had proper tracking in place. The experiment ran for several weeks with proper statistical rigor.

Decision Metric

Activation % = % of new users who perform at least one of the following key actions in their first month of Pro usage:

  • Data Exports
  • Target List creation
  • Benchmarks usage
  • Alerts setup
  • Saved Search
  • LinkedIn Icon Click

6 actions chosen to reflect most used and general features across the platform

Conclusions & Analysis

1. Checklist Effect: Validated ‚úì

Is exposed to checklist against not: Activation outperformance by 50% (23% vs 35%, 35% is quite on the usual baseline for this KPI)

  • Users in Treatment: 827 (60%)
  • Users in Control: 537

Conclusion: Providing new users with a checklist of recommended actions significantly improves activation rates. The 50% lift demonstrates clear value in guided onboarding.

2. Personalization Effect: Strong Signal, Needs Validation

Is exposed to personalised checklist against default checklist:

  • Activation outperformance by c.50% against default checklist (30% vs 43%, 43% is quite high compared to the usual baseline for this KPI)
  • Activation outperformance by 119% against no checklist at all / usual product experience (20% vs 43% activation percentage)
  • Users in Treatment, 'Fundraising users': 310 (37%)
  • Users in Control, 'Not fundraising users': 517

‚ö†Ô∏è Important Nuance: Potential Fundraising User Bias

The personalization effect was only tested on fundraising users. There could be a 'fundraising users' bias‚Äîmaybe historically, all users 'which are here for fundraising' have had better activation.

Mitigation strategy: I am building a 'user segmentation proxy' dataset to be able to, given users pageviews, guess what they are here for, and having this information could help us better protect ourselves against these bias.

Still, it agrees with common sense that a more personalised feature is more impactful than a generic one. And, releasing this feature will help us gather such information for every new user, with no negative impact seen from the feature so far.

Statistical Rigor

The analysis followed proper statistical procedures:

  • Welch t test, robust to any particular assumptions about the data
  • Quick analyses confirmed no substantial differences in the two groups, ensuring their main difference was the product they were exposed to
  • Statistical significance achieved with alpha = 5% (we are okay with having a 5% chance of concluding there is an impact when there is not)

Recommendation & Next Steps

Release the new feature to all new users

The data supports a full rollout. We will gather very useful data from new users that helps us help them, while improving activation rates.

With the baseline improvement validated, the focus shifts to:

  • Collecting use case information from all users to normalize for potential fundraising effect
  • Building user segmentation proxy dataset for better bias detection
  • Refining the personalization logic based on user behavior patterns across all segments
  • Monitoring long-term retention impact to validate the activation-retention correlation
