What Problem Did This Solve?

At Indeed Flex, a marketplace connecting hospitality venues with gig workers, I identified a critical gap in our data stack: understanding user behavior was essential but painfully slow. Product managers and analysts faced several challenges:

  • Days-long wait times for simple data pulls across 8+ different datasets
  • Analyst frustration spending days on tedious, repetitive data work instead of high-value analysis
  • PM frustration unable to quickly answer critical questions: What makes a user retained? What drives lifetime value?
  • Slow iteration cycles on user retention and LTV analysis

The core question: What makes a user retained? What drives revenue for life? We needed answers in seconds, not days.

What I Built: The User Intelligence Platform

I identified this gap and drove the creation of a centralized user intelligence platform that transformed how the team understood user behavior.

Platform architecture: From 8+ datasets to one centralized intelligence layer

How It Works

Using DBT (Data Build Tool) and CI/CD pipelines, I consolidated 8+ disparate datasets into a single, clean, well-documented central table that provided:

  • User behavior tracking: Complete app activity history
  • Retention metrics: 3-month, 6-month, 9-month retention rates
  • LTV analysis: Lifetime value calculations and predictions
  • Feature usage patterns: Which features drive retention and upsells
  • Cohort analysis: Easy segmentation and comparison

Clean, modular SQL architecture spanning 30+ tables and 800+ lines of code

Technical Implementation

The platform was built with modern data engineering best practices:

  • DBT: Modular, testable SQL transformations
  • Snowflake: Scalable data warehouse
  • CI/CD: Automated testing and deployment
  • Documentation: CEO-friendly business documentation of all tables and metrics
  • Code quality: 30+ tables, 800+ lines of clean, maintainable SQL

Extract of SQL code showing technical depth and code quality

How It Could Be Even Better

Looking forward, the platform could be enhanced with:

Semantic Layer Implementation

Instead of speaking in column names, users could speak in business terms: "Select 3 Month Retention metric" or "Select 9 month LTV"

Modern enablers: AI and embeddings make this easier to implement today, providing even more intuitive self-service access.

Key Learnings

  • Identify the gap: I saw the frustration and time waste, and took initiative to solve it
  • Consolidation creates value: From messy to simple, accurate, and insightful
  • Self-service empowers teams: When data is accessible, everyone moves faster
  • Documentation matters: CEO-friendly docs ensure platform adoption and trust
  • Time savings compound: Every second saved multiplies across the entire team
